GitHub Pages ログ管理サイト構想整理

📅 2026-04-18

GitHub Notion 自動化
ログ管理WebページをGitHub Pagesで構築する構想を整理。分類・検索・スマホ対応を軸に設計。第1段階の骨格実装に着手。

背景・目的

これまでローカルのPC内に散在していたログや資料を、Web上で一元管理できる仕組みを構築したいと考えてきた。 スマホからもアクセスできること、あとから分類・検索ができること、を最低条件として設計を進めた。

Notionやローカルフォルダでの管理も試みたが、PCとスマホの行き来や、人に見せるときの手間が課題だった。 GitHub Pagesを使えばURLで共有でき、静的サイトなのでコストもかからない。

方針

以下の構成で進めることを決定した。

  • 表示: GitHub Pages（静的HTML）
  • データ管理: JSON形式（posts.json）
  • 検索・絞り込み: バニラJavaScript（フレームワーク不要）
  • 分類体系: 大分類 / 中分類 / タグ / 年度の4軸
  • 閲覧端末: PC・スマホ両対応（レスポンシブ）

実装優先順位

一気に作り切るより「動く骨格を作って実データを入れながら調整する」方針。 分類の粒度やタグの設計は、実際に10〜30件溜めてみないと最適解が見えない。

  • 第1段階: トップ一覧、大分類フィルター、詳細ページ表示
  • 第2段階: キーワード検索、タグ・中分類の絞り込み
  • 第3段階: 最近の更新表示、関連記事、人気タグ

今後の課題

posts.jsonを手で書くのは20件を超えたあたりから苦行になる。 Markdownのフロントマターから自動生成するPythonスクリプトを早めに整備する予定。

また、中分類「自動化」が「仕事」と「HINOVA」両方に存在するように、 分類が重複するケースが今後出てくる。運用しながら調整する。
